01 · BATCH ELT

WarehouseLab

Raw events → analytics-ready marts via an orchestrated, tested ELT pipeline.
raw→ dbt staging→ dbt marts→ Metabase · orchestrated byDagster

A deterministic generator lands raw events in Postgres. dbt cleans and types them into staging, then computes business marts — daily active users, revenue by day, a user dimension. Dagster runs the DAG on a daily schedule; Metabase reads the marts. The whole thing materializes from empty with make run.

02 · REAL-TIME

StreamPulse

Process live events and compute real-time windowed metrics with alerting.
producer→ Redpanda→ processor→ metrics→ dashboard+alerter

A producer streams synthetic events into a Redpanda topic. A stream processor maintains tumbling windows — events per minute, conversion rate per 5-minute window — and writes them to a metrics store. A live dashboard renders them; an alerter fires when a metric crosses a threshold. The windowing logic is pure and unit-tested to exact aggregates.

03 · DATA QUALITY

DataGuard

Check datasets for schema, null, range, freshness & drift; score and alert.
expectations→ check engine→ scores→ FastAPI→ Streamlit

A pluggable engine runs checks — schema, null, range, freshness, and distribution drift — against Postgres tables on a schedule. Drift is detected with PSI (primary) and KS (secondary); results roll up into a deterministic per-table quality score. A FastAPI service serves the results; a Streamlit dashboard renders scores, failing checks, and trends. Every check is a pure, tested function.
